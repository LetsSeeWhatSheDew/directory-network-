rendering code consume specific columns from the response.

NOTE on the current 500 error
-----------------------------------------------
The Vercel log shows a Postgres 42703 error mentioning `products_or_ser...`.
Error 42703 is "undefined column" (not "undefined table"), which suggests
the `products_or_services` table exists but is missing one of the columns
referenced in the order clause — most likely `category` or `name`.

Because the page uses `Promise.all([getHours, getAttributes, getProducts])`
and `fetchJson` throws on non-ok, a single missing column in any of the
three tables crashes the entire listing page.
